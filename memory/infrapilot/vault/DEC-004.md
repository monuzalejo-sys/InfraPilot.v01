# DEC-004 — ORION model policy: adaptive by difficulty, ceiling Opus

**title:** ORION model policy: adaptive by difficulty, ceiling Opus

**description:** The orchestrator picks each agent's model by task/step difficulty via the Agent tool's model param: trivial->haiku, normal->sonnet, hard->opus (ceiling Opus is allowed; floor Haiku). The orion-analyst rates difficulty per sub-objective/step and those ratings drive builder/verifier/fixer model choice; builders run one-per-step so a single plan can mix haiku and opus steps in parallel. Agent frontmatter defaults were set (analyst/planner/builder/verifier/fixer = sonnet, reflector = haiku) so a naive spawn doesn't inherit the main conversation's model. Full rubric lives in the '1b. Model selection' section of ~/.claude/skills/orion/SKILL.md (mirrored in runtime/skills/).

**reason:** User chose adaptive-by-difficulty with an Opus ceiling. This environment has no live token metering (RFC-0004 budgets are conceptual here), so model choice is the only real cost lever — spending should track difficulty instead of defaulting everything to the most expensive model.
