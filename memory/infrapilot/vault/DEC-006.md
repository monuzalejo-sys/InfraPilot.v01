# DEC-006 — AMM schema embedded in reflector agent, not runtime-parsed

**title:** AMM schema embedded in reflector agent, not runtime-parsed

**description:** Instead of the reflector loading knowledge-objects.ts at runtime to validate/parse the AMM schema, the schema is baked into the agent's SYSTEM prompt as a table + constraints (LIFETIME_TO_TIER, status values, type-specific fields). This eliminates a runtime TS parser dependency, reduces latency (no file I/O per reflection), and makes the schema version explicit per run (reflector inherits it from its frontmatter at agent load time). If the schema evolves, the reflector's SYSTEM prompt is updated; no AMM-runtime process exists to bump a version field.

**reason:** Session 2026-07-02 verified that a hand-emulated schema (no live runtime) is viable for ORION's current scale. Baking the schema into the reflector is safer and simpler than attempting to maintain a separate parser binary.
