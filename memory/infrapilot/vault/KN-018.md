# KN-018 — ORION agent-spawn reliability observed 2026-07-05: 3 of 10 spawns this session d

**fact:** ORION agent-spawn reliability observed 2026-07-05: 3 of 10 spawns this session died mid-work (2 hit session/context limits, 1 got a connection-closed error) while leaving partial-but-real work on disk. Recovery protocol used successfully: diff the dead agent's claimed file ownership against actual `git status`/working-tree state, then either (a) finish the remaining delta inline in the orchestrator when small, or (b) re-spawn a narrowly-scoped agent targeting only the unfinished files. Do not blindly re-spawn the same broad task — it duplicates already-completed work and wastes tokens.
