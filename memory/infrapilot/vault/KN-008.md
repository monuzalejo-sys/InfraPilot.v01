# KN-008 — infrapilot-app's tax-by-currency table is hardcoded in lib/utils.ts with standar

**fact:** infrapilot-app's tax-by-currency table is hardcoded in lib/utils.ts with standard legal rates: COP and MXN use IVA @19% and @16% respectively; PEN uses IGV @18%; fallback is 'Impuesto' @18%. This branching is correct for the current 3-currency deployment (COP primary), but if InfraPilot expands to additional countries or wants to pull rates from an external source of truth (tax authority APIs, a database, etc.), the helpers (taxLabel(), defaultTaxRate()) must be refactored to consume dynamic configuration instead of literal values.

**context:** Extracted from PEND-006 F3 (Tax rate mismatch) resolution. The hardcoded rates are intentional and correct per currency, but the pattern (literal switch on currency) is brittle if country coverage expands.

**source:** session 2026-07-02, PEND-006 caveat
