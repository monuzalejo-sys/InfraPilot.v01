# KN-022 — Supabase migration pattern (error.code 42P01 / /does not exist|schema cache/) fo

**fact:** Supabase migration pattern (error.code 42P01 / /does not exist|schema cache/) for all future migrations (003+): when an API route attempts to query a table that doesn't exist because a later migration hasn't run, the route detects error.code === '42P01' or error message contains /does not exist|schema cache/ and responds HTTP 503 {error:'MIGRATION_NNN_REQUIRED'}. The client's components/migration-notice.tsx catches 503 responses and displays a clear banner. This pattern was established for MIGRATION_002 (suppliers_quotes table) and must be reused for 003+ to gracefully degrade and inform users of pending migrations without breaking the app.

**context:** Built 2026-07-06 to support the new /proveedores feature (supplier management + price comparatives). Reduces user friction when migrations haven't been applied yet.

**source:** session 2026-07-06, infrapilot-app commit 131040f (app/api/*, components/migration-notice.tsx, supabase/migrations/002_suppliers_quotes.sql)
