# KN-021 — lib/disciplines.ts is the canonical registry of InfraPilot's engineering discipl

**fact:** lib/disciplines.ts is the canonical registry of InfraPilot's engineering disciplines: an array of DisciplineConfig objects (id, label, modules, categories, units). The sidebar's main nav derives from getNavSections(), which iterates this array. Adding a new engineering discipline (e.g., 'estructura') requires editing ONLY lib/disciplines.ts: append to the array with id/label/modules/categories/units. Current disciplines: core (construcción, includes new /proveedores feature), topografia, licitaciones, analisis. The sidebar reactivity + routing is already generic and will auto-pick up new entries.

**context:** Established pattern session 2026-07-06 when adding the new Proveedores module to the core discipline. The registry avoids scattered config and makes discipline discovery declarative.
