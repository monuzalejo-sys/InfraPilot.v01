# KN-014 — React 19 StrictMode failure pattern: the 'lazy guard' idiom in a component body 

**fact:** React 19 StrictMode failure pattern: the 'lazy guard' idiom in a component body (if (!started) { setStarted(true); void load(); }) fires setState synchronously during render, before mount, triggering an async side effect outside an effect boundary -> console error under StrictMode's double-invoke. Fix: move the guard into a useEffect (mount effect) instead of inline render logic. Project convention adopted: where the lazy-guard-in-render pattern is intentional and already effect-wrapped, suppress react-hooks/set-state-in-effect with a scoped eslint-disable-next-line + comment explaining why, rather than restructuring further. Two separate builders reproduced this same mistake independently this session (build:restyle-datos, build:restyle-flujo) before the orchestrator fixed it inline — treat this as a known trap when assigning React 19 StrictMode work to builders.

**context:** Found while resolving PEND-007 (6 set-state-in-effect warnings) during the 2026-07-05 full-app redesign.
