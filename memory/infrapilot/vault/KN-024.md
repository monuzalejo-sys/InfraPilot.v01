# KN-024 — SheetJS/xlsx round-trip discovery: the xlsx library silently DISCARDS formula ce

**fact:** SheetJS/xlsx round-trip discovery: the xlsx library silently DISCARDS formula cells {t:'n', f} if they carry NO pre-computed `v` value when the .xlsx binary is written. Workaround verified: every cell with f:'formula' MUST carry v:precalculated_numeric_result. lib/excel-export.ts implements this as function f(formula, value, fmt) that always bundles the precalculated value. Export pattern for modules: lib/<module>-export.ts exports crearLibroConFormulas(hojas, nombreArchivo) as a utility — clients can consume this for editable, formula-driven exports without relearning SheetJS internals. Pattern: every new module that exports Excel MUST use crearLibroConFormulas and populate all formula cells via f(formula, value).

**context:** Discovered during ROAD-001 (hub + Excel exports). Verified via round-trip test: XLSX.write → XLSX.read confirms formulas persist only when v is present. This is a silent data-loss bug in the xlsx library if the precalculated value is omitted.

**source:** session 2026-07-06, build:visual (dashboard hub with Excel export, opus)
