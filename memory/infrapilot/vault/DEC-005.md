# DEC-005 — ORION autocommits on verification PASS; push stays manual

**title:** ORION autocommits on verification PASS; push stays manual

**description:** Standing policy: when an ORION task passes VERIFYING, autocommit the changes in whichever repo changed (app code -> infrapilot-app submodule / InfraPilot.Ai; memory & docs -> parent InfraPilot.v01), with a clear message. Do NOT push (terminal push blocked, KN-001) — list pending commits for the user to push via GitHub Desktop. Escalated/failed tasks are not auto-committed.

**reason:** User chose 'autocommit al pasar verificación' so work is checkpointed automatically without losing the manual-push safety gate.
