# KN-007 — AMM Schema invariant: tier MUST match lifetime per RFC-0002 §5.2. Tier (Working/

**fact:** AMM Schema invariant: tier MUST match lifetime per RFC-0002 §5.2. Tier (Working/Project/Permanent) is a Kubernetes-like cardinality constraint: Working items are Session-scoped; Project items are Sprint/Project lifetime; Permanent items are Permanent lifetime. The validator (tools/validate-memory.mjs) caught real violations in session 2026-07-02: KN-001..005 initially had lifetime=Permanent tier=Project (invalid); repair was KN-001→tier Permanent (machine-level fact justifies permanence); KN-002..005→lifetime Project (facts about infrapilot-app project, not eternal truths). Lesson: future architects must validate tier⟷lifetime pairing on every memory write.

**context:** Discovered during validation of the E2E pipeline; zero-dep Node.js validator runs offline and caught violations that would have persisted indefinitely if not automated.
