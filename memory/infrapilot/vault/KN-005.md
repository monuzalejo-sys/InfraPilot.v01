# KN-005 — infrapilot-app's next build was failing because the Groq client was instantiated

**fact:** infrapilot-app's next build was failing because the Groq client was instantiated at module scope in app/api/cotizar/route.ts and app/api/licitaciones/analyze/route.ts, causing the build to fail when GROQ_API_KEY was not in the environment. FIXED by moving client instantiation into the request handlers (after the missing key guard) and committed as 5efdde3. Pattern: never instantiate API-key-dependent clients at module scope in Next route files — construct them inside the handler after validating the key is present.

**context:** This was F2 in the audit: classified as REAL BUG (build-breaking). Deletion of 6 mock-*.ts files (F5, ~2,650 lines removed) would have failed the build until this was fixed.

**source:** session 2026-07-01, orion-analyst audit + orion-fixer fix + orion-verifier PASS (commit 5efdde3)
