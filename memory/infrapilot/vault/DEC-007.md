# DEC-007 — Proportional lifecycle: ceremony scales with task size

**title:** Proportional lifecycle: ceremony scales with task size

**description:** Standing policy (SKILL.md 1d): trivial tasks (1 file, fully specified) run fully inline with zero agent spawns; small tasks (2-4 files) use at most one builder plus a spawned verifier; only substantial multi-step work pays the full analyst->planner->builders->verifier->reflector pipeline. Reflection (memory update) is mandatory at every level, inline or spawned. Rationale: the E2E test showed the full pipeline costs ~10x inline execution for small artifacts (~168k subagent tokens for a 91-line README).

**reason:** User asked for minimum token consumption per task; full-pipeline-by-default was the single largest waste. Verified 2026-07-02: this session's improvement batch ran inline at a small fraction of the E2E cost.
